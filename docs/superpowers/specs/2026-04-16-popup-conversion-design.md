# Дизайн: попапы для повышения конверсии лендинга

**Дата:** 2026-04-16  
**Проект:** sunction-stylist-day (cd.sunction.store)  
**Статус:** Одобрен

---

## Контекст и проблема

Лендинг — регистрация на «День стилиста» 19 апреля 2026 г. (12 слотов, 8 свободных).  
Трафик: платный (таргет), аудитория холодная.  
Проблема: пользователи кликают на «Забронировать место», скроллят к форме, но **форму не отправляют**.

Две основные причины отказа:
1. **Недоверие** — не хотят оставлять телефон незнакомому бренду
2. **Недостаточно убеждены** — нет финального толчка к действию

---

## Решение: два компонента

### Компонент A — Exit-intent popup (десктоп)

**Цель:** поймать пользователя в момент ухода и закрыть сделку прямо в попапе.

**Триггер:**
- Событие `mouseleave` с `clientY < 10` (мышь двигается к адресной строке / вкладкам)
- Срабатывает **один раз за сессию** (флаг `sunction_popup_shown` в `sessionStorage`)
- Не срабатывает если `localStorage.sunction_submitted === 'true'`

**Содержимое:**
- Живой countdown до `2026-04-19T12:00:00+03:00` — формат «3 дн. 4 ч. 22 мин.»
- Индикатор мест: пульсирующий зелёный dot + «Осталось 8 мест из 12»
- Trust-сообщение: «Запись бесплатная. Телефон — только для подтверждения времени. Спам не отправляем.»
- Inline-форма: поля Имя + Телефон (с маской), чекбокс согласия (ссылка на privacy.html), кнопка «Забронировать место» (disabled до согласия)
- После успешной отправки — success-state (аналогичен основной форме на странице)

**UX:**
- Закрытие: кнопка × в правом верхнем углу + клик по тёмному overlay
- Анимация открытия: `fade-in` + `translateY(20px → 0)` за 0.3s ease
- Overlay: `rgba(0,0,0,0.55)`, z-index выше всего контента

**Стиль:**
- Фон попапа: `#FFFFFF`, border-radius: 16px, max-width: 440px
- Шрифты и цвета — идентичны основному лендингу (Cormorant + Nunito Sans, --bg-dark, --border)
- Countdown: Cormorant serif, крупный

---

### Компонент B — Trust nudge у формы (все устройства)

**Цель:** в момент колебания над формой снять страх перед оставлением телефона и добавить срочность.

**Триггер:**
- Пользователь скроллит в зону видимости секции `#signup` (IntersectionObserver, threshold 0.3)
- Провёл в ней **8 секунд** без отправки формы
- Не показывается если форма уже отправлена (`localStorage.sunction_submitted`)
- Показывается только **один раз** (`sessionStorage.sunction_nudge_shown`)

**Содержимое:**
- Иконка замка (🔒 или SVG)
- Текст: «Телефон нужен только для подтверждения времени. До мероприятия — [N дн.], осталось 8 мест.»
- Кнопка закрытия ×

**UX:**
- Появляется slide-down над полями формы (внутри `.signup__form`, перед `<form>`)
- Скрывается при нажатии × или после успешной отправки формы

**Стиль:**
- Фон: `var(--bg-dark)` (#1A1A1A), текст: #FFFFFF
- border-radius: 10px, padding: 14px 16px
- Анимация: `max-height: 0 → auto` + `opacity: 0 → 1` за 0.3s

---

## Общая логика (shared JS)

```
function getCountdown(targetISO) → { days, hours, minutes, seconds }
localStorage key: 'sunction_submitted'   // true после любой отправки формы
sessionStorage key: 'sunction_popup_shown'  // показан ли exit popup
sessionStorage key: 'sunction_nudge_shown'  // показан ли nudge
```

При отправке любой формы (основной, в попапе) — выставляется `sunction_submitted = 'true'` и скрываются все компоненты.

## Аналитика (Yandex.Metrika)

| Событие | Goal name | Когда |
|---|---|---|
| Exit popup показан | `exit_popup_show` | При открытии попапа |
| Форма в попапе отправлена | `exit_popup_submit` | После успешного submit в попапе |
| Trust nudge показан | `trust_nudge_show` | При появлении плашки |

---

## Ограничения

- Exit-intent работает **только на десктопе** — на мобиле `mouseleave` не триггерится. Trust nudge (B) — основной инструмент для мобильной аудитории.
- Счётчик «8 мест» — хардкод. Если нужна динамика, потребуется отдельный бэкенд.
- Все изменения только в `index.html` — новых файлов не создаётся.
