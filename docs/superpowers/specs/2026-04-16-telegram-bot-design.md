# Telegram Bot — запись на День стилиста

**Дата:** 2026-04-16
**Событие:** День стилиста, 19 апреля, Земляной Вал 7, Москва

## Цель

Добавить возможность записаться на событие через Telegram-бота как дополнительный канал рядом с существующей HTML-формой. Форма остаётся нетронутой; записи через оба канала попадают в одну Google Sheets.

## Компоненты

### 1. Кнопка на лендинге (`index.html`)

В секции записи (`#signup`) добавляется вторая кнопка под существующей формой:

```
[Забронировать место]     ← существующая форма
[Записаться через Telegram] → t.me/<botusername>
```

Кнопка открывает бота в новой вкладке (`target="_blank"`). Стиль — вторичная кнопка (outlined), чтобы не конкурировать с основной формой.

### 2. Google Apps Script — бот-обработчик

Новый GAS-скрипт (отдельный от текущего `form-handler`), деплоится как Web App с доступом `Anyone`.

**Функции:**
- `doPost(e)` — точка входа для Telegram webhook
- `handleUpdate(update)` — роутинг по типу входящего сообщения
- `sendMessage(chatId, text, replyMarkup)` — отправка сообщения через Telegram Bot API
- `writeToSheet(name, phone)` — запись строки в Google Sheets

**Состояние диалога** хранится в `PropertiesService.getUserProperties()` по ключу `state_<chatId>`. Значение — JSON-объект `{ step, name, phone }`.

Шаги (`step`):
- `idle` — начальное состояние
- `awaiting_name` — ждём имя
- `awaiting_phone` — ждём телефон
- `awaiting_confirm` — ждём подтверждение

**Безопасность:** в URL webhook добавляется секретный параметр `?token=<SECRET>`. В `doPost` проверяется наличие токена перед обработкой.

### 3. Диалог бота

```
/start
  → "Привет! Записываю вас на День стилиста 19 апреля. Как вас зовут?"
  step: awaiting_name

[пользователь вводит имя]
  → "Отлично, {имя}! Укажите номер телефона:"
  step: awaiting_phone

[пользователь вводит телефон]
  → "Проверьте данные:\nИмя: {имя}\nТелефон: {телефон}\nВсё верно?"
     Inline-кнопки: [✅ Да] [🔄 Начать заново]
  step: awaiting_confirm

[callback: Да]
  → запись в Google Sheets
  → "Вы записаны! Ждём вас в субботу, 19 апреля 🎉\nАдрес: Земляной Вал, 7"
  step: idle

[callback: Начать заново]
  → "Хорошо, начнём сначала. Как вас зовут?"
  step: awaiting_name
```

Любое сообщение в состоянии `idle` (не `/start`) → "Напишите /start чтобы записаться."

### 4. Google Sheets

Записи через бота добавляются в ту же таблицу, что и заявки через форму. Дополнительная колонка `Источник` со значением `telegram` (у записей через форму — `form`).

Колонки: `Timestamp | Имя | Телефон | Источник`

> Примечание: существующие строки из формы не имеют колонки «Источник» — нужно добавить заголовок вручную или в скрипте формы.

## Последовательность настройки

1. Получить новый токен бота у BotFather (`/revoke` → `/token`)
2. Создать GAS-скрипт, добавить токен и ID таблицы в `Script Properties`
3. Задеплоить GAS как Web App, скопировать URL
4. Зарегистрировать webhook: открыть в браузере `https://api.telegram.org/bot<TOKEN>/setWebhook?url=<GAS_URL>?token=<SECRET>`
5. Добавить кнопку на лендинг, указать `t.me/<botusername>`
6. Проверить сквозной сценарий: пройти запись в боте, убедиться что строка появилась в таблице

## Не входит в scope

- Уведомления в Telegram при новой записи
- Отмена или изменение записи через бота
- Антиспам / лимит записей
- Telegram Mini App
